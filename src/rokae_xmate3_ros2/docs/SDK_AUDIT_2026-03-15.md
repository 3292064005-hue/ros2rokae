# xCore SDK V2.1 对照审计与修复记录

审计对象：`rokae_xmate3_ros2`（ROS2 Humble + Gazebo 11）

对照依据：`xCore_SDK_C++使用手册V2.1_A.pdf`。

## 审计方法

1. 以手册第 4 章 / 第 8 章列出的 SDK API 为准，整理出对外兼容方法清单。
2. 对 `include/rokae/sdk_shim.hpp` 逐项 grep 检查方法是否存在。
3. 对存在但语义不完整或行为明显偏离手册的实现做补齐。
4. 在无法做真机闭环的 Gazebo 环境下，优先补齐“接口一致 + 仿真可运行 + 示例可执行”的实现。

## 审计结论

- 手册中当前包已暴露的 SDK 风格方法名：**全覆盖**。
- 本次修复的关键缺口集中在：
  - 坐标系标定 `calibrateFrame(...)`
  - 实时碰撞阈值 `setCollisionBehaviour(...)`
  - 已弃用但仍保留入口的 `automaticErrorRecovery(...)`
  - 实时循环 `setFinished()` 结束语义
  - `useStateDataInLoop=true` 的循环内状态刷新
  - `jointAcc_c` / `tauVel_c` 的实时状态估计
  - `disconnectNetwork()` 的“立即停止并关闭状态接收”行为

## 本次代码修复

### 1. 实时状态缓存

- 新增统一的 `detail::refresh_state_cache(...)` 刷新逻辑
- 优先使用 `getRtJointData()` 获取实时关节状态，失败时回退到普通查询接口
- 新增：
  - `jointAcc_c`：由关节速度差分得到
  - `tauVel_c`：由关节力矩差分得到
- `startReceiveRobotState()` 现在会预热一帧缓存
- `stopReceiveRobotState()` / `disconnectNetwork()` 会清空缓存并复位导数状态

### 2. 实时循环行为

- `setControlLoop()` 现在会尊重 `Command::setFinished()` 标记
- `startLoop(false)` 修复了原先先置位再 `stopLoop()` 导致线程立即退出的问题
- `useStateDataInLoop=true` 时，循环开始前会自动刷新状态缓存

### 3. 实时接口补齐

- 新增 `RtMotionControlCobot::setCollisionBehaviour(...)`
- 新增 `MotionControl<MotionControlMode::RtCommand>::automaticErrorRecovery(...)`
- `disconnectNetwork()` 现在会停止循环、停止运动，并清除状态接收缓存

### 4. 坐标系标定

- `Robot_T::calibrateFrame(...)` 不再直接返回 `not_supported`
- Gazebo 下实现了一个**几何近似版**：
  - `FrameType::tool`：使用多姿态法兰位姿做 TCP 平移求解
  - `FrameType::wobj/base/world`：使用三点建系近似恢复坐标系
- 标定结果会同步回写到当前 session 的 `toolset_cache`

## 仍需明确的仿真限制

以下能力已经给出兼容入口，但仍属于仿真近似，而非真机控制器等价行为：

- 直接力矩控制与阻抗控制的真实闭环
- 碰撞检测阈值与真机安全链路的一致性
- `calibrateFrame(...)` 的控制器级高精度标定结果
- 实时网络行为（真实 1 kHz 以太网链路）

这些限制来自 Gazebo/ROS2 仿真边界，不是接口缺失。
